Getting Started with AVA-Gen¶

This guide summarizes the minimum steps needed to get AVA-Gen running for local development. For more detail, see the other docs in this folder.

1. Install dependencies¶

From the project root:

pip install .

This installs the ava-gen CLI and the runtime components.

2. Configure environment¶

Copy the example configuration and edit it:

cp .env.example .env

Set at least:

  • OPENAI_API_KEY â€“ your OpenAI API key.
  • (Optionally) AVA_GEN_OPENAI_MODEL, AVA_GEN_WORKSPACE_ROOT, AVA_GEN_RUNTIME_DATA_DIR.

3. Run the CLI¶

See docs/cli.md for the full pipeline. A typical flow:

ava-gen prepare <app_id> path/to/MyTests.java
ava-gen pipeline <app_id>

This populates your workspace with extracted tests, VA methods, skill descriptions, and intent artifacts.

4. Start the runtime server¶

Once you have generated skills, intents, and action plans, you can start the runtime server:

uvicorn runtime.api.server:app --reload

The runtime API surface and conversation flow are described in docs/runtime_design.md and docs/intent_validation.md.
